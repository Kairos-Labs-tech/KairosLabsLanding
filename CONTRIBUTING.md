# Contributing

## Workflow
1. Open an issue to discuss significant changes.
2. Branch off main (e.g., eat/feature-name, ug/bug-name).
3. Commit using [Conventional Commits](https://www.conventionalcommits.org/).
4. Open a Pull Request.

## Expectations
- **Code Quality**: Ensure tests pass and the code is formatted.
- **Documentation**: If you change behavior, update the related docs.
- **Review**: Provide clear PR descriptions to speed up reviews.
